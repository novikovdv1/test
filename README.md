## Выбор инструмента

Groq API был выбран по следующим причинам:
- Бесплатные API Keys с рейт лимитами которые я указал в analyzer.py
- llama-3.1-8b-instant- одна из лучших открытых моделей для работы с русским языком
## Ограничения
1.Для работы с Groq API требуется ускоритель интернета,иначе ошибка Exception: API Error: 403, {"error":{"message":"Forbidden"}}.
(API ключ  являются условно бесплатным, предназначенными только для ознакомительных целей)
2. Модель может пропускать некоторые неявные возражения или темы, которые обсуждаются косвенно
3. Отсутствие специализированного обучения на данных компании может приводить к менее точной классификации специфичных для бизнеса ситуаций

## Следующие шаги

1. Fine-tuning модели на внутренних данных компании для улучшения точности распознавания тем и возражений
2. Разработка API и интеграция с существующими системами (CRM, мессенджеры)
3. Внедрение системы мониторинга качества анализа и механизма обратной связи от менеджеров

## Запуск проекта

1. Установите зависимости:
```bash
pip install -r requirements.txt
```

2. Создайте файл .env и добавьте ваш API ключ:
```
GROQ_API_KEY=
```

3. Запустите анализатор:
```bash
python analyzer.py
```
